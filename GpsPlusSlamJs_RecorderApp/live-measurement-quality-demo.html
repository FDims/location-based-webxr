Quick start
How to use this demo

This page is a standalone simulation of the live measurement loop. It shows how uncertainty, baseline, and confirmation evolve as more observations arrive.

  • Start with Run Far Replay Demo to watch a distant point get re-marked from multiple spots.
  • When the baseline is small or the fit is still weak, the prompt should say move_sideways.
  • As you add better views, the uncertainty should shrink, the sphere should tighten, and the prompt should become ready_to_confirm.
  • Use confirmRequested and then confirmSucceeded to commit the point once it is ready.
  • Use the sliders and reducer buttons to test specific states manually.
Reducer events
  • observationAdded: recompute the current draft from the live inputs.
  • removeLastRay: back up one observation and force the draft to re-evaluate.
  • confirmRequested: ask the reducer to enter the pending confirm step when the point is ready.
  • confirmSucceeded: commit the point and mark it confirmed.
  • confirmFailed: simulate a failed commit and return to coaching.
  • retargetDraft: start a fresh provisional target, clearing stale readiness.
  • cancelDraft: discard the draft and return to idle.
  • resetAfterConfirmFailed: recover from a failed confirm and clear the draft state.
Demo scripts
  • Run Near Demo: a nearby target that quickly becomes ready and confirms.
  • Run Far Replay Demo: the main replay path for a far target; baseline grows, uncertainty shrinks, and the prompt changes from coaching to confirmation.
  • Run Noisy Demo: shows how the state reacts when one observation gets worse before recovering.
  • Stop Script: cancel the currently running scripted replay immediately.
Evaluation reasons

The reason tags in the list below are the raw outputs from the pure quality check. They explain why the state is ready, blocked, or still refining.

  • Hard blockers like no_solved_point, solver_degenerate, stale_observation, and insufficient_inliers mean the draft cannot be confirmed yet.
  • Signal buckets like u_0.6, b_1.0, r_0.8, and i_1.0 summarize uncertainty, baseline, residual, and inlier strength.
  • entered_ready means the score crossed into ready; exited_ready means it dropped back out; steady means the quality state stayed in place.

Tip: The demo is most useful when you keep this panel open the first time, then close it once you know the controls.
